# Timeline / Dock Operating Context

## Core policy

- Original input files must not be deleted.
- Input and output locations should be fixed by default.
- Paths may be configurable, but arbitrary path handling should not be the normal workflow.
- Analysis and file organization should be separated.
- Rename / organize operations require manifest records.
- CLI / worker / output contract are the source of truth.
- Web UI is an operator console.
- Docker / docker compose / start scripts should be reproducible.
- Local-first operation is preferred.
- Do not hide gaps, assumptions, or transformation loss.

## Standard concepts

```text
InputSource = configured input source
InputItem   = detected processable input from InputSource
Job         = processing request based on an InputItem snapshot and profile
Run         = one execution attempt for a Job
Artifact    = output generated by a Run
```

## Fixed directory baseline

```text
data/
  config/
  input/
  output/
  state/
  logs/
```

Recommended environment variables:

```text
TIMELINE_WORKSPACE=./data
TIMELINE_INPUT_ROOT=./data/input
TIMELINE_OUTPUT_ROOT=./data/output
TIMELINE_STATE_ROOT=./data/state
```

## Output baseline

```text
data/output/runs/
  <product-name>/
    <job-id>/
      job.json
      input_snapshot.json
      run_001/
        run.json
        status.json
        manifest.json
        result.json
        fidelity_report.json
        logs/
        artifacts/
```

## Approval required

Ask before:

- deleting raw input
- mass moving raw input
- mass renaming raw input
- destructive migration
- output contract breaking change
- repo or product rename
- deploy
- cloud apply / destroy
- external posting
- secret changes
- broad send-all execution without dry-run

## Allowed without confirmation

You may perform:

- repository investigation
- docs updates
- AGENTS.md updates
- `.env.example` updates
- non-destructive small code changes
- local tests
- smoke tests
- Docker build / compose smoke tests
- localhost bind fixes
- fixed input/output documentation
